# Контрольные вопросы по C# (ООП и свойства)

## Часть 1. Классы, объекты, конструкторы, методы

### 1. Определение класса
**Класс** — это шаблон (чертёж) для создания объектов, объединяющий данные (поля) и поведение (методы, свойства). Класс описывает состояние и поведение объектов, которые будут созданы на его основе.

### 2. Синтаксис объявления класса

```csharp

{
    // поля
    // конструкторы
    // методы
    // свойства
}
csharp
public class Person
{
    private string name;   // поле
    public Person() { }    // конструктор
    public void SayHello() // метод
    {
        Console.WriteLine("Hello");
    }
}

5. Понятие конструктора класса
Конструктор — это специальный метод класса, который автоматически вызывается при создании экземпляра. Имя конструктора совпадает с именем класса, он не возвращает значение (даже void). Служит для инициализации полей и выполнения начальных действий.

6. Синтаксис объявления конструктора
csharp
[модификатор_доступа] ИмяКласса([параметры])
{
    // тело конструктора
}
csharp
public class Person
{
    public string Name;
    
    public Person(string name)  // конструктор с параметром
    {
        Name = name;
    }
}
7. Конструктор по умолчанию
Конструктор по умолчанию — это конструктор без параметров. Если в классе явно не объявлен ни один конструктор, компилятор автоматически создаёт пустой конструктор по умолчанию, который инициализирует поля значениями по умолчанию (0, null, false). Если же вы объявили любой конструктор с параметрами, конструктор по умолчанию автоматически не создаётся.

### 12. Модификаторы доступа private и public, их назначение
- **public** (открытый) — член класса доступен из любого другого кода. Используется для создания интерфейса класса.
- **private** (закрытый) — член класса доступен только внутри того же класса. Используется для сокрытия внутренней реализации (инкапсуляция).

По умолчанию члены класса имеют доступ `private`.

### 13. Правила именования полей и методов в C#
**Методы:**
- Глагол или глагольная фраза (Run, GetName, SaveToFile)
- Стиль PascalCase (каждое слово с заглавной буквы)

**Поля:**
- Публичные поля (public) — также PascalCase (FirstName, Age)
- Приватные поля (private) — camelCase (первое слово с маленькой буквы), часто с префиксом `_` (`_name`, `_age`)
- Константы (const) — обычно PascalCase или UPPER_CASE

**Общие синтаксические правила:**
- Имена не могут совпадать с ключевыми словами (если нужно — используйте `@class`)
- Имя может начинаться с буквы или подчёркивания `_`, далее буквы, цифры, подчёркивания
- Регистр важен (Name и name — разные идентификаторы)

## Часть 2. Свойства (Properties)

### 1. Понятие свойства
Свойство — это член класса, который предоставляет гибкий механизм для чтения, записи или вычисления значения частного поля. Свойства выглядят для внешнего кода как поля, но внутри содержат специальные методы доступа (аксессоры) `get` и `set`, позволяющие реализовать контроль, проверки или вычисления.

### 2. Синтаксис объявления свойства
[модификаторы_доступа] тип ИмяСвойства
{
get { return поле; }
set { поле = value; }
}

text

С проверкой:
private int age;
public int Age
{
get { return age; }
set { if (value > 0) age = value; }
}

text

### 3. Преимущества использования свойств
- Инкапсуляция — скрывают внутреннее представление данных
- Контроль доступа — валидация, логирование, ленивая инициализация в `get`
- Гибкость — можно сделать свойство только для чтения или только для записи
- Совместимость — замена публичного поля на свойство не ломает клиентский код
- Удобство — свойства поддерживаются в привязке данных (WPF) и сериализации

### 4. Создание свойств только для чтения или записи
**Только для чтения (read-only):** есть только `get`
public string Name { get; } // автосвойство
// или классическое
private string _name;
public string Name => _name; // выражение-тело

text

**Только для записи (write-only):** есть только `set` (встречается редко)
private string _secret;
public string Secret
{
set { _secret = value; }
}

text

### 5. Что такое вычисляемые свойства? Их назначение
**Вычисляемое свойство** — свойство, которое не хранит значение, а вычисляет его «на лету» на основе других полей или свойств.

**Назначение:** предоставить удобный доступ к производным данным без хранения избыточной информации.
public double Width { get; set; }
public double Height { get; set; }
public double Area => Width * Height; // вычисляемое свойство (только get)

text

### 6. Автоматические свойства, синтаксис автоматических свойств
**Автоматическое свойство** — сокращённая форма, когда компилятор автоматически создаёт скрытое поле и стандартную реализацию get/set.
public string Name { get; set; } // чтение и запись
public int Age { get; } // только для чтения
public string Address { get; set; } = "Moscow";// с инициализацией
public int Count { get; private set; } // сеттер только внутри класса

text

### 7. Преимущества автосвойств
- Меньше кода — не нужно явно объявлять приватное поле
- Удобство рефакторинга — легко заменить на полноценное свойство с логикой
- Инициализация — поддерживают инициализаторы прямо в объявлении
- Разные модификаторы — можно указать, например, `public get` и `private set`
- Участие в интерфейсах — автосвойства допустимы в интерфейсах (C# 8.0+)

> Примечание: автосвойства не подходят, если в аксессорах нужна дополнительная логика (валидация, события, вычисления). В таких случаях используется классическое свойство с явным полем.
